# DECISIONS

주요 설계 결정을 "배경 / 대안 / 결정 이유" 형식으로 최신순으로 기록한다.

## 2026-07-02 — 탭 기반 단일 페이지 네비게이션 (라우터 미도입)

**배경**
대시보드·거래·예산 세 화면을 오가야 하는데, 개발 초기에는 세 페이지가 한 화면에 세로로 나열돼 있어 탐색이 불편했다. 화면 전환 방식을 정해야 했다.

**대안**
- `react-router-dom` 도입 후 URL 라우팅(`/dashboard`, `/transactions`, `/budget`)
- `App.tsx`에서 `useState`로 활성 탭을 관리하는 단순 조건부 렌더링
- 기존처럼 한 페이지에 전부 세로로 나열

**결정 이유**
`useState` 기반 탭 전환을 선택했다. 화면이 3개뿐이고 서버·로그인 없이 브라우저에서 동작하는 개인용 SPA라 URL 라우팅/딥링크 요구가 없어, 라우터 의존성을 추가하는 비용이 이득보다 크다. 전부 나열하는 방식은 스크롤이 길고 맥락이 섞여 제외했다. 추후 딥링크나 뒤로가기 지원이 필요해지면 이 지점에 `react-router-dom`을 도입한다.
