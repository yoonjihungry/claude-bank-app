# 가계부 웹앱 구현 계획

> 기술 스택, 폴더 구조, 데이터 모델, 아키텍처 규칙은 CLAUDE.md 참조.

## Context
`claude-bank-app` 디렉토리는 현재 비어 있는 새 프로젝트다. 사용자는 개인용 가계부(수입/지출 기록·통계) 웹앱을 처음부터 만들고자 한다.

## 이번 작업 범위
핵심 기능: 거래 CRUD, 카테고리별/월별 통계 차트, 월별 예산 설정 및 초과 표시, 기간·카테고리·금액 필터.

## 초기 셋업
- `npm create vite@latest . -- --template react-ts`
- 의존성 설치: `recharts`, `date-fns`, `uuid`

## 구현 로직 세부사항
> 상태 관리·저장소 계층 원칙은 CLAUDE.md의 Architecture Rules 참조. 아래는 이번 기능들의 구체적 계산 로직.

- **통계 계산** (`useStatistics`): 선택 월 기준으로 카테고리별 지출 합계, 총수입/총지출/잔액, 예산 대비 사용률을 `useMemo`로 파생 계산.
- **예산 초과 판정**: 카테고리별 `사용액 / limit` 비율 계산 → 80% 경고(노랑), 100% 초과(빨강) 색상 표시.
- **필터**: `FilterBar` 상태(기간·카테고리·최소/최대 금액)를 `useTransactions`에서 조합해 리스트 필터링.

## 구현 순서
1. Vite 프로젝트 초기화 + 의존성 설치
2. 타입·기본 카테고리·저장소 계층 작성
3. `LedgerContext` (reducer + localStorage 연동)
4. 거래 입력 폼 + 목록 (TransactionsPage) — CRUD 동작 확인
5. 요약 카드 + 차트 (DashboardPage)
6. 필터 바
7. 예산 설정 + 초과 표시 (BudgetPage)
8. 레이아웃/탭 네비게이션 + 스타일 정리

## 검증 방법
- `npm run dev` 실행 후 브라우저에서:
  - 수입/지출 거래 추가·수정·삭제 → 목록 및 요약 카드 즉시 반영 확인
  - 페이지 새로고침 후 데이터 유지 확인 (localStorage 영속화)
  - 대시보드에서 카테고리 파이/월별 막대 차트가 입력 데이터와 일치하는지 확인
  - 예산 한도 설정 후 지출 추가 시 80%/100% 경고 색상 전환 확인
  - 필터(기간·카테고리·금액)로 목록이 정확히 좁혀지는지 확인
- `npm run build`로 타입 에러 없이 빌드되는지 확인
